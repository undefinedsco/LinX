# 统一凭据管理器设计方案

> 讨论日期：2024-12-03
> 状态：设计讨论，非当前 MVP

## 概述

构建一个统一的个人凭据管理器，类似 1Password/Bitwarden，但数据存储在用户自己的 Solid Pod 中。

## 凭据类型

| 类别 | 示例 | 关键字段 |
|------|------|----------|
| **账号密码** | 网站登录、邮箱 | 用户名、密码、网址 |
| **API 密钥** | OpenAI、云服务 | Key、Endpoint、供应商 |
| **OAuth 令牌** | Google、GitHub | Token、Refresh、Scope |
| **SSH/证书** | 服务器、代码签名 | 公钥、私钥、指纹 |
| **银行卡** | 信用卡、储蓄卡 | 卡号、有效期、CVV |
| **身份证件** | 身份证、护照 | 号码、有效期 |
| **备注/文本** | 序列号、WiFi密码 | 纯文本 |

## 数据模型

通用 schema + 类型字段：

```typescript
{
  id: string
  type: 'login' | 'api_key' | 'card' | 'identity' | 'note' | 'ssh' | ...
  name: string
  provider?: string       // 供应商标识
  icon?: string           // 自定义图标或自动匹配
  fields: Field[]         // 动态字段，根据 type 有默认模板
  tags?: string[]
  folderId?: string
  favorite?: boolean
  createdAt: Date
  updatedAt: Date
  lastUsedAt?: Date
}
```

## UI 布局

### 整体结构：左侧筛选器 + 右侧智能表格

```
┌──────────────┬─────────────────────────────────────────────────────────────┐
│ 📋 全部       │ 🔍 搜索凭据...                          [+ 粘贴或输入]      │
│ ⭐ 收藏       │ ──────────────────────────────────────────────────────────── │
│ 🕐 最近       │                                                             │
│ ────────────│  🤖 API 密钥 (3)                                    ▼ 收起   │
│ 🏷️ 标签       │ ──────────────────────────────────────────────────────────── │
│   工作       │  供应商      名称       Key            Endpoint    更新时间  │
│   个人       │  OpenAI     生产环境   sk-••••9x2a    默认        3天前     │
│ ────────────│  OpenAI     测试环境   sk-••••b3f2    自定义      1周前     │
│ 📦 类型       │  + 添加 API 密钥...                                         │
│   登录       │                                                             │
│   API密钥    │  🔐 登录 (2)                                       ▼ 收起   │
│   银行卡     │ ──────────────────────────────────────────────────────────── │
│              │  供应商      名称       用户名          网址        更新时间  │
│              │  Google     主邮箱     test@gmail     google.com  2周前     │
│              │  GitHub     工作号     zhangsan       github.com  1月前     │
│              │  + 添加登录...                                              │
└──────────────┴─────────────────────────────────────────────────────────────┘
```

### 设计决策

1. **左侧是筛选器，不是数据分组**
   - 全部、收藏、最近
   - 标签（用户自定义）
   - 类型（系统预设）

2. **右侧按类型分区展示**
   - 每种类型有自己的列字段
   - 可折叠收起
   - 每区块有独立添加入口

3. **智能表格优势**
   - 信息密度高，一屏看几十条
   - 可排序/筛选
   - 批量操作支持
   - 内联编辑

### 添加方式

两个入口：

| 入口 | 位置 | 交互 |
|-----|------|-----|
| **[+ 粘贴或输入]** | 顶部搜索旁 | 弹出输入框，智能识别 |
| **+ 添加xxx...** | 每个分区末尾 | 内联展开编辑 |

### 智能识别

用户粘贴内容，自动识别类型和供应商：

| 输入内容 | 识别为 | 自动填充 |
|---------|-------|---------|
| `sk-...` | OpenAI API | 供应商 + Key |
| `ghp_...` | GitHub Token | 供应商 + Token |
| `AKIA...` | AWS Key | 供应商 + Key |
| `xxx@gmail.com` | Google 登录 | 供应商 + 用户名 |
| `https://github.com/xxx` | GitHub 登录 | 供应商 + URL |
| 无法识别 | 让用户选类型 | — |

### 空状态

首次进入默认选中"全部"，表格区域显示引导：

```
                    🔐

         粘贴任意凭据，自动识别存储

    ┌─────────────────────────────────┐
    │ sk-xxx... / user@mail.com / ... │
    └─────────────────────────────────┘

        数据加密存储在你的 Solid Pod
```
